GUÍA DE ESTRUCTURA DEL PROYECTO GUSMUSS DIAMOND
==================================================

Hola. Este documento es para tu uso privado, como pediste, para que puedas consultar la estructura técnica exacta de tu proyecto. (Este archivo NO se subirá a GitHub).

El proyecto Gusmuss Diamond está dividido en **dos bloques principales** (Arquitectura Desacoplada):

1. EL BACKEND (Symfony / PHP) - La carpeta raíz
--------------------------------------------------
Es el motor de nuestra tienda (la API REST). Contiene la lógica profunda, consulta la base de datos y provee de datos crudos al Frontend.

- `/src/Entity` -> Aquí viven las clases de la base de datos (Product, User, Order).
- `/src/Controller/Api` -> Son los "Controladores". Aquí definimos las rutas web invisibles (endpoints) para que React pueda pedir datos. Por ejemplo: `ProductController.php` atiende las peticiones de ropa, y `AuthController.php` maneja logins y registros.
- `/src/Controller/Admin` -> Controladores exclusivos para el panel de administración de EasyAdmin (para crear/eliminar ropas desde la trastienda).
- `/config/packages` -> Aquí configuramos comportamientos clave del Backend, como la seguridad de contraseñas (`security.yaml`), la conexión a la Base de Datos (`doctrine.yaml`) y el sistema de JWT.


2. EL FRONTEND (React / Vite) - Dentro de la carpeta `/frontend/`
--------------------------------------------------
Es el "escaparate visual". Consume la información que el backend le pasa, y pinta la web y sus interacciones bonitas.

- `/frontend/src/pages/` -> Cada archivo representa una pantalla entera de tu web (Home.jsx, Collection.jsx, Contacto.jsx, Cart.jsx, Login.jsx).
- `/frontend/src/components/ui/` -> Piezas de código reutilizables y pequeñas, como Botones, el SearchBar, o la famosa ImageGallery.
- `/frontend/src/components/layout/` -> Componentes estructurales (el Navbar de arriba o el Footer de abajo) que envuelven a la página.
- `/frontend/src/context/` -> Archivos con lógica global compartida entre páginas: 
  - `AuthContext.jsx` (recuerda y comparte globalmente si estamos logueados).
  - `CartContext.jsx` (recuerda y comparte la cesta de la compra).
  - `LanguageContext.jsx` (recuerda el idioma ES o ENG elegido).
- `/frontend/src/i18n/` -> (De Internationalization). Aquí vive `translations.js`, el diccionario con absolutamente todo el texto estático de la página en Inglés y Español.


LA RESPUESTA A TU DUDA: "Cómo compruebo lo del mail de verificación"
--------------------------------------------------
Como estamos en "modo desarrollo", no queremos bombardear correos reales. Hemps configurado Symfony usando un interceptor virtual de correos llamando `MailHog`.

Paso a paso para ver los correos:
1. Abre una pestaña nueva en tu navegador.
2. Escribe la ruta de tu buzón local de MailHog: `http://localhost:8025/`
3. Al entrar verás una bandeja de entrada virtual (como si fuera Gmail).
4. Cuando registras una "cuenta nueva de prueba", Symfony envía el email a este buzón. Haz clic en él y verás el precioso botón de "Verificar mi correo".
5. Si haces clic en el botón de verificar de ese correo, viajarás de vuelta a la tienda local y activará tu cuenta en la base de datos al 100%.

(Nota: Si la IP de tu Mailhog u otro contenedor es distinta, consulta los puertos en tu herramienta de Docker Desktop).
